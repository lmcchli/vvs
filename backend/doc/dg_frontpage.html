 Introduction

This document contains the javadoc documentation about the backend API. An html version of this document can be found at the backend module publishing area.

The backend API contains some of the modules that were developed in the MAS component to access MUR, MS and send traffic events (to MER and NTF). It also contains configuration and logging functionality.

 Modules

Currently the API contains the following modules:

·         Configuration Manager

·         Logging

·         Profile manager

·         Mailbox

·         Traffic event sender

·         External component register

·         Media object

·         Message sender

 Function Description

For detailed description on how each component (module) works see FS and FD for the specific module. These documents can currently be found on the foundation artifact page.

 Build and Run information

In order to build and run a program that uses the backend API all libraries that comes with the API must be included in the class path.

The configuration files logmanager.xml, backend.xml and trafficevents.xml and their corresponding schema definition files (backend.xsd and trafficevents.xsd) must be located somewhere so that the logmanager component can read them. See also documentation for the LogManager class.

The junit tests should all pass when run from ant, if run from IDE some manual steps are needed:

·         Locate the junit-target in build.xml

·         Copy the appropriate (check the copy section to see if the profilemanager or javamail version should be used) mailhost.properties file to somewhere in your classpath.
